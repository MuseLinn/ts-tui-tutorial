# TS-TUI-Tutorial 架构设计文档

> 一个基于全屏沉浸式 TUI 的交互式 TypeScript 学习工具  
> 版本：v1.0 | 日期：2026-04-15 | 作者：Sisyphus

---

## 1. 项目概述与目标

### 1.1 项目愿景
打造一个运行在终端里的、全屏沉浸式的交互式教学应用，引导**零基础（甚至连 JavaScript 都不太熟）**的用户从零开始掌握 TypeScript 开发。应用通过理论讲解、代码示例高亮、实时互动测验和"修复类型错误"的练习，让学习者在终端内完成完整的编程学习闭环。

### 1.2 核心目标
- **零配置开箱即用**：用户只需安装并运行一个 CLI 命令即可开始学习
- **全屏沉浸式体验**：非问答流，而是类似 IDE 的分栏布局，支持代码高亮、导航栏、状态栏
- **即时反馈**：所有练习均集成 TypeScript 编译器 API，实时给出类型错误反馈
- **渐进式解锁**：课程按难度分层，掌握前置内容后方可解锁后续章节
- **断点续学**：自动保存学习进度，关闭终端后仍可无缝续上

## 2. 目标受众画像

| 维度 | 描述 |
|------|------|
| **编程基础** | 几乎为零，可能不了解变量、函数、条件语句等基本概念 |
| **JS 基础** | 无或极其薄弱，需要从 JS 语法与运行逻辑讲起 |
| **TS 期望** | 希望系统性地掌握 TS，最终能看懂并写出带类型的代码 |
| **学习习惯** | 偏好动手实践、需要即时正向反馈、注意力持续时间较短 |

**设计启示**：
- 课程必须**从 JS 基础切入**，将 TS 的类型系统作为"给 JS 代码增加安全说明"的工具来讲解，而不是单独抽象地教类型论
- 每节课时长控制在 **3-5 分钟**
- 理论讲解必须配合可运行的代码示例和至少一个互动环节

## 3. 技术栈选型

### 3.1 核心框架：Ink (React for CLI)

**选择理由**：
- **TypeScript 原生**：`create-ink-app --typescript` 直接生成 TS 项目，与教学主题完全一致
- **全屏持久化 TUI**：基于 Yoga Flexbox 布局引擎，可实现侧边栏+主内容区+底部状态栏的复杂界面
- **React 组件化**：`<Lesson>`、`<Quiz>`、`<CodeBlock>`、`<Exercise>` 等教学内容天然对应 React 组件
- **生态丰富**：`@inkjs/ui` 提供 Select/TextInput/ProgressBar，`ink-syntax-highlight` 可做代码高亮
- **生产级验证**：被 Claude Code、Gemini CLI、Prisma CLI、Shopify CLI 采用

### 3.2 类型检查引擎：TypeScript Compiler API (`typescript`)

**选择理由**：
- 无需外部依赖，直接调用 `ts.createProgram()` + `ts.getPreEmitDiagnostics()` 即可在内存中完成类型检查
- 与课程教学内容版本一致，不会出现 tsc 版本差异导致的困惑
- 可自定义 `ts.CompilerOptions`，对新用户隐藏复杂配置

### 3.3 其他依赖

| 包名 | 用途 |
|------|------|
| `@inkjs/ui` | 菜单选择、文本输入、进度条、确认框 |
| `ink-syntax-highlight` | 终端内语法高亮 |
| `chalk` | 终端颜色输出（Ink 内部已依赖，可直接复用） |
| `zod` | 进度数据与配置文件的 schema 校验 |
| `fuse.js` | 课程搜索（未来扩展） |

### 3.4 开发工具

| 工具 | 用途 |
|------|------|
| `tsx` | 开发时直接运行 TS 源码 |
| `eslint` + `@typescript-eslint` | 代码规范 |
| `prettier` | 格式化 |
| `vitest` | 单元测试（类型检查引擎、课程数据解析等） |

## 6. 课程与练习数据模型

### 6.1 课程结构（Lesson）

```typescript
interface Lesson {
  id: string;                    // 唯一标识，如 "01-variables"
  title: string;                 // 课程标题
  description: string;           // 简短描述
  tier: 'beginner' | 'intermediate' | 'advanced';
  steps: LessonStep[];           // 课程内部步骤序列
  prerequisites: string[];       // 前置课程 ID 列表
  estimatedMinutes: number;      // 预计学习时长
}

type LessonStep = 
  | TheoryStep 
  | CodeDemoStep 
  | QuizStep 
  | ExerciseStep;

interface TheoryStep {
  type: 'theory';
  content: string;               // Markdown 格式文本
}

interface CodeDemoStep {
  type: 'code-demo';
  code: string;                  // TypeScript 代码
  focusLines?: number[];         // 需要高亮聚焦的行号
  explanation: string;           // 代码逐行解释
}

interface QuizStep {
  type: 'quiz';
  question: string;
  options: { label: string; value: string; isCorrect: boolean; explanation?: string }[];
  allowMultiple: boolean;
}

interface ExerciseStep {
  type: 'exercise';
  instructions: string;
  initialCode: string;           // 初始（带错误的）代码
  expectedCode?: string;         // 期望答案（用于提示系统）
  validationMode: 'tsc' | 'exact-match' | 'contains';
  hints: string[];               // 分阶段提示
  compilerOptions?: ts.CompilerOptions; // 本练习特定的 tsc 配置
}
```

### 6.2 进度数据结构（Progress）

```typescript
interface UserProgress {
```

## 7. 课程体系设计（针对零基础）

由于目标受众**连 JS 都不太熟**，课程设计不能假设学员知道变量、函数、循环等概念。我们将把 JS 基础与 TS 类型注解**融合教学**，用 TS 的类型系统来帮助理解 JS 的行为。

### 7.1 课程总览（约 30-35 节课，分 3 个阶段）

#### Stage 1：JS + TS 基础入门（约 12 课）
从"变量是什么"开始，逐步建立"给数据贴标签"的直觉。

| 课号 | 标题 | 核心概念 |
|------|------|----------|
| 01 | 变量与类型注解 | `let`/`const`，基本类型 `string`/`number`/`boolean` |
| 02 | 运算符与表达式 | 算术、比较、逻辑运算，结果的类型 |
| 03 | 条件判断 | `if/else`，条件表达式的 `boolean` 类型 |
| 04 | 函数基础 | 函数定义、参数、返回值，为参数加类型注解 |
| 05 | 数组与元组 | `number[]`、`string[]`、固定长度元组 `[string, number]` |
| 06 | 对象类型入门 | `{ name: string; age: number }`，点号访问属性 |
| 07 | 循环结构 | `for`、`while`，循环变量的类型推断 |
| 08 | Interface 初识 | 用 `interface` 给对象形状命名 |
| 09 | 可选属性与默认值 | `?` 标记、函数默认参数 |
| 10 | 类型别名 type | `type` 与 `interface` 的对比与选择 |
| 11 |  any 与 unknown | 不安全与安全的"任意类型" |
| 12 | 阶段综合练习 | 修复一个小项目的类型错误 |

#### Stage 2：理解类型的威力（约 12 课）
引入联合类型、 narrowing、泛型等核心能力，让用户能处理更复杂的代码。

| 课号 | 标题 | 核心概念 |
|------|------|----------|
| 13 | Union Types（联合类型） | `string \| number` |
| 14 | Narrowing 基础 | `typeof`、相等判断缩小类型范围 |
| 15 | 字面量类型 | `'success' \| 'error'`、`as const` |
| 16 | 函数类型深入 | 回调函数类型、箭头函数注解 |
| 17 | 泛型入门 | `<T>`、泛型函数、数组泛型 `Array<T>` |
| 18 | 泛型约束 | `extends` 限制泛型参数 |
| 19 | 接口继承与实现 | `extends`、`implements` |
| 20 | Type Guards | 自定义类型保护函数 `is` |
| 21 | 工具类型（上） | `Partial`、`Required` |
| 22 | 工具类型（下） | `Pick`、`Omit`、`Record` |
| 23 | 异步与 Promise | `Promise<T>`、`async/await` 的类型 |
| 24 | 阶段综合练习 | 重构一段 JS API 代码为强类型 TS |

#### Stage 3：进阶与工程化思维（约 8 课）
为想要继续深入的用户准备。

| 课号 | 标题 | 核心概念 |
|------|------|----------|
| 25 | 模块与导入导出 | `import type`、`export type`、ESM 基础 |
| 26 | 索引签名与映射 | `{ [key: string]: number }` |
| 27 | 类型推导与 typeof | `typeof` 获取变量类型、`ReturnType` |

### 7.2 每节课的内部结构

每节课严格遵循 **"三段式"**：
1. **Theory (理论)**：1-2 屏文本 + 1 个代码示例
2. **Quiz (测验)**：2-3 道选择题/判断题，巩固概念理解
3. **Exercise (练习)**：1 道"修复类型错误"的代码题（Rustlings 风格）

## 8. 实时类型检查引擎设计（TscService）

### 8.1 核心设计原则
- **内存中运行**：不在磁盘生成 `.js` 文件，不产生临时文件污染
- **毫秒级反馈**：用户修改代码后，0.5s 内给出诊断结果
- **错误信息友好化**：将 tsc 原始错误翻译成更贴近初学者的语言

### 8.2 技术实现

使用 `typescript` 包的 `createProgram` + `getPreEmitDiagnostics`：

```typescript
import ts from 'typescript';

class TscService {
  private defaultOptions: ts.CompilerOptions = {
    target: ts.ScriptTarget.ES2022,
    module: ts.ModuleKind.CommonJS,
    strict: true,
    noEmit: true,           // 关键：不生成文件
    skipLibCheck: true,
  };

  check(code: string, overrideOptions?: ts.CompilerOptions): Diagnostic[] {
    const fileName = 'virtual:///exercise.ts';
    const options = { ...this.defaultOptions, ...overrideOptions };
    
    // 自定义 compilerHost，从内存读取源代码
    const host = ts.createCompilerHost(options);
    const originalGetSourceFile = host.getSourceFile.bind(host);
    
    host.getSourceFile = (name, target) => {
      if (name === fileName) {
        return ts.createSourceFile(name, code, target);
      }
      return originalGetSourceFile(name, target);
    };
    
    host.readFile = (name) => {
      if (name === fileName) return code;
      return ts.sys.readFile(name);
    };
    
    host.fileExists = (name) => {
      if (name === fileName) return true;
      return ts.sys.fileExists(name);
    };

    const program = ts.createProgram([fileName], options, host);
    const diagnostics = ts.getPreEmitDiagnostics(program);
    
    return diagnostics.map(d => this.friendlyDiagnostic(d));
  }
  
  private friendlyDiagnostic(d: ts.Diagnostic): Diagnostic {
    const message = ts.flattenDiagnosticMessageText(d.messageText, '\n');
    return {
      line: d.file ? d.file.getLineAndCharacterOfPosition(d.start!).line + 1 : 0,
      column: d.file ? d.file.getLineAndCharacterOfPosition(d.start!).character + 1 : 0,
      message: this.translate(message), // 将 TS2345 等翻译成中文友好提示
      code: d.code,
    };
  }
}
```

### 8.3 校验模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| `tsc` | 仅要求代码通过 TypeScript 类型检查 | 大多数"修复类型错误"练习 |
| `exact-match` | 代码不仅要无错，还要与期望代码完全一致 | 检查特定语法掌握情况 |
| `contains` | 代码中必须包含某些关键字/模式（正则匹配） | 检查是否使用了 `interface` 或泛型等 |

## 9. 进度持久化与状态管理

### 9.1 状态管理方案

采用 **React Context + useReducer**，因为应用状态相对集中，不需要引入 Redux/Zustand 等额外复杂度。

```typescript
interface AppState {
  // 导航状态
  currentLessonId: string;
  currentStepIndex: number;
  currentView: 'theory' | 'quiz' | 'exercise';
  
  // 交互状态
  userCode: string;            // 当前练习中用户编辑的代码
  selectedQuizOptions: string[];
  showHint: boolean;
  hintIndex: number;
  
  // 反馈状态
  diagnostics: Diagnostic[];
  lastResult: 'idle' | 'correct' | 'incorrect';
  
  // 全局数据
  progress: UserProgress;
  settings: AppSettings;
}
```

### 9.2 自动保存策略

- **事件触发保存**：用户完成一节课、提交一次练习、或按 `q` 退出时自动写入 `progress.json`
- **定时保存**：每 30 秒静默保存一次（后台写入，不阻塞 UI）
- **崩溃恢复**：应用启动时读取 `progress.json`，恢复到上次离开的位置

### 9.3 解锁规则

```typescript
function isLessonUnlocked(lesson: Lesson, progress: UserProgress): boolean {
  if (lesson.prerequisites.length === 0) return true;
  return lesson.prerequisites.every(id => 
    progress.completedLessons[id]?.score >= 60
  );
}
```

## 12. 风险与备选方案

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| **Ink 在 Windows 终端兼容性** | 中 | 使用 Windows Terminal 或 VS Code 集成终端测试；Ink 对主流终端支持良好 |
| **tsc 内存检查在大型代码上的性能** | 低 | 练习题代码通常 <50 行，不存在性能问题 |
| **初学者对全屏 TUI 的接受度** | 中 | 提供清晰的键盘提示；首次启动时展示 10 秒引导动画 |
| **课程内容的教学效果** | 高 | 参考 Rustlings 和 Total TypeScript 的课程顺序；MVP 后根据用户反馈迭代 |
| **多行代码编辑体验差** | 中 | 第一阶段使用"填空"或"单处修改"模式，降低编辑复杂度；后续考虑引入外部编辑器调用 |
